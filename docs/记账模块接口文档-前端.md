# 记账模块接口文档（前端）

**说明**：所有 `/bill/**` 接口需登录，请求时携带 Session（Cookie 中 JSESSIONID）。未登录返回 `code: 401`。  
**统一响应结构**：`{ "code": 200|400|401|500, "message": "提示语", "data": ... }`

---

## 一、记账分类（bill_category）

### 1.1 新增记账分类

- **请求**：`POST /bill/category/add`
- **请求体**（JSON）

```json
{
  "categoryName": "餐饮"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| categoryName | string | 是 | 分类名称，1–100 字；同一用户下不可重复 |

- **成功响应**（200）

```json
{
  "code": 200,
  "message": "分类添加成功",
  "data": {
    "categoryId": 1,
    "categoryName": "餐饮"
  }
}
```

- **失败示例**：`categoryName` 为空 → 400，`message`: "分类名称不能为空"；重名 → 400，"分类名称已存在"

---

### 1.2 编辑记账分类

- **请求**：`PUT /bill/category/update`
- **请求体**（JSON）

```json
{
  "categoryId": 1,
  "categoryName": "新餐饮"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| categoryId | number | 是 | 分类 id，须为当前用户的分类 |
| categoryName | string | 是 | 新名称，1–100 字；同用户下不可与其他分类重名 |

- **成功响应**（200）

```json
{
  "code": 200,
  "message": "分类编辑成功",
  "data": {
    "categoryId": 1,
    "categoryName": "新餐饮"
  }
}
```

---

### 1.3 删除记账分类

- **请求**：`DELETE /bill/category/delete`
- **请求体**（JSON）

```json
{
  "categoryId": 1
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| categoryId | number | 是 | 要删除的分类 id |

- **成功响应**（200）

```json
{
  "code": 200,
  "message": "分类删除成功",
  "data": null
}
```

- **失败示例**：该分类下有收支记录 → 400，`message`: "该分类下有收支记录，无法删除"；分类不存在 → 400，"分类不存在"

---

### 1.4 查询用户所有记账分类

- **请求**：`GET /bill/category/list`
- **请求体**：无
- **成功响应**（200）

```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    { "categoryId": 1, "categoryName": "餐饮" },
    { "categoryId": 2, "categoryName": "交通" }
  ]
}
```

列表按创建时间升序。

---

## 二、收支记录（bill_record）

### 2.1 新增收支记录

- **请求**：`POST /bill/record/add`
- **请求体**（JSON）

```json
{
  "categoryId": 1,
  "type": 2,
  "amount": 35.50,
  "remark": "午餐",
  "createTime": "2026-03-15 12:00:00"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| categoryId | number | 是 | 分类 id，须为当前用户的分类 |
| type | number | 是 | 1=收入，2=支出 |
| amount | number | 是 | 金额，>0，最多两位小数 |
| remark | string | 否 | 备注 |
| createTime | string | 否 | 记账时间，格式 `yyyy-MM-dd HH:mm:ss`；不传则用当前时间 |

- **成功响应**（200）

```json
{
  "code": 200,
  "message": "收支记录添加成功",
  "data": {
    "recordId": 1,
    "amount": 35.50
  }
}
```

- **失败示例**：分类不存在 / 非当前用户 → 400；type 非 1/2 → 400；amount≤0 → 400，"金额必须大于0"

---

### 2.2 编辑收支记录

- **请求**：`PUT /bill/record/update`
- **请求体**（JSON）

```json
{
  "recordId": 1,
  "categoryId": 2,
  "type": 2,
  "amount": 40.00,
  "remark": "晚餐",
  "createTime": "2026-03-15 18:00:00"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| recordId | number | 是 | 要编辑的记录 id |
| categoryId | number | 否 | 分类 id，须为当前用户的分类 |
| type | number | 否 | 1=收入，2=支出 |
| amount | number | 否 | 金额，>0 |
| remark | string | 否 | 备注 |
| createTime | string | 否 | 格式 `yyyy-MM-dd HH:mm:ss` |

- **成功响应**（200）

```json
{
  "code": 200,
  "message": "收支记录编辑成功",
  "data": {
    "recordId": 1,
    "amount": 40.00
  }
}
```

- **失败示例**：记录不存在 / 无权限 → 400

---

### 2.3 删除收支记录

- **请求**：`DELETE /bill/record/delete`
- **请求体**（JSON）

```json
{
  "recordId": 1
}
```

- **成功响应**（200）

```json
{
  "code": 200,
  "message": "收支记录删除成功",
  "data": null
}
```

---

### 2.4 查询最近收支记录

- **请求**：`GET /bill/record/recent`
- **Query 参数**

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| limit | number | 否 | 10 | 条数 |

示例：`GET /bill/record/recent?limit=20`

- **成功响应**（200）

```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "recordId": 1,
      "categoryId": 1,
      "categoryName": "餐饮",
      "type": 2,
      "amount": 35.50,
      "remark": "午餐",
      "createTime": "2026-03-15T12:00:00"
    }
  ]
}
```

`createTime` 为 ISO 格式（LocalDateTime 序列化）。按 `createTime` 降序。

---

### 2.5 按时间范围查询收支记录

- **请求**：`GET /bill/record/listByTime`
- **Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| startTime | string | 是 | 开始时间，格式 `yyyy-MM-dd HH:mm:ss` |
| endTime | string | 是 | 结束时间，格式 `yyyy-MM-dd HH:mm:ss` |

示例：`GET /bill/record/listByTime?startTime=2026-03-01%2000:00:00&endTime=2026-03-31%2023:59:59`

- **成功响应**（200）：格式同 2.4 的 `data` 数组，按 `createTime` 降序。

```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "recordId": 2,
      "categoryId": 1,
      "categoryName": "餐饮",
      "type": 2,
      "amount": 50.00,
      "remark": "晚餐",
      "createTime": "2026-03-15T18:00:00"
    }
  ]
}
```

## 三、月度预算（bill_budget）

### 3.1 设置 / 修改月度预算

- **请求**：`POST /bill/budget/set`
- **请求体**（JSON）

```json
{
  "yearMonth": "2026-03",
  "budgetAmount": 2000.00
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| yearMonth | string | 是 | 年月，格式 `yyyy-MM` |
| budgetAmount | number | 是 | 月度总预算，>0 |

- **成功响应**（200）

```json
{
  "code": 200,
  "message": "预算设置成功",
  "data": {
    "yearMonth": "2026-03",
    "budgetAmount": 2000.00
  }
}
```

若该月已有预算则更新，否则新增。

---

### 3.2 查询月度预算及超支情况

- **请求**：`GET /bill/budget/get`
- **Query 参数**

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| yearMonth | string | 否 | 当前月 | 格式 `yyyy-MM` |

示例：`GET /bill/budget/get?yearMonth=2026-03`

- **成功响应**（200）

```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "yearMonth": "2026-03",
    "budgetAmount": 2000.00,
    "totalSpend": 800.50,
    "remainAmount": 1199.50,
    "isOverspend": false
  }
}
```

| 字段 | 说明 |
|------|------|
| budgetAmount | 该月预算金额（未设置则为 0） |
| totalSpend | 该月支出合计（type=2） |
| remainAmount | 剩余预算（预算 - 总支出，不低于 0） |
| isOverspend | 是否超支：总支出 > 预算 |

---

## 四、数据可视化（图表）

### 4.1 按周 / 月查询收支趋势

- **请求**：`GET /bill/chart/trend`
- **Query 参数**

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| timeType | string | 否 | month | `month` 或 `week` |
| yearMonth | string | 按需 | - | timeType=month 时必填，格式 `yyyy-MM` |
| weekStart | string | 按需 | - | timeType=week 时必填，格式 `yyyy-MM-dd` |
| weekEnd | string | 按需 | - | timeType=week 时必填，格式 `yyyy-MM-dd` |

示例：  
- 按月：`GET /bill/chart/trend?timeType=month&yearMonth=2026-03`  
- 按周：`GET /bill/chart/trend?timeType=week&weekStart=2026-03-10&weekEnd=2026-03-16`

- **成功响应**（200）

```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    { "date": "2026-03-10", "income": 0.00, "spend": 35.50 },
    { "date": "2026-03-11", "income": 0.00, "spend": 50.00 }
  ]
}
```

按天汇总：`date` 为日期，`income` 为当日收入合计，`spend` 为当日支出合计；范围内无记录的天数也会返回，金额为 0。

---

### 4.2 按类别查询支出占比

- **请求**：`GET /bill/chart/categoryRatio`
- **Query 参数**

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| yearMonth | string | 否 | 当前月 | 格式 `yyyy-MM` |

示例：`GET /bill/chart/categoryRatio?yearMonth=2026-03`

- **成功响应**（200）

```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    { "categoryName": "餐饮", "totalAmount": 500.00, "ratio": 62.34 },
    { "categoryName": "交通", "totalAmount": 300.00, "ratio": 37.66 }
  ]
}
```

| 字段 | 说明 |
|------|------|
| categoryName | 分类名称 |
| totalAmount | 该月该分类支出合计 |
| ratio | 占比（%），保留两位小数，该分类支出 / 该月总支出 × 100 |

仅统计支出（type=2）；该月无支出时 `data` 为空数组。

---

## 五、错误码与提示

| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 参数或业务错误（如必填为空、金额≤0、分类重名、该分类下有记录无法删除等） |
| 401 | 未登录 |
| 500 | 系统异常 |

错误时 `data` 一般为 `null`，具体原因看 `message`。
